### 1.1 技術棧

| 層級 | 技術 | 版本/說明 |

## 第四部分：API 介面

### 4.1 盤點 API
- `POST /inventory/record/submit`: 提交單筆盤點 (含 WebP 照片/GPS/備註)。支援即時上傳與批次同步。
- `POST /inventory/plan/<id>/close`: 結案並觸發複盤/調整邏輯。
- `GET /inventory/asset/fetch/<asset_id>`: 查詢資產基本資訊 (供掃描後即時回饋)。

### 4.2 前端離線機制
- **IndexedDB**: `AssetInventoryDB` 資料庫，`pending_records` 物件儲存。
- **WebP 壓縮**: Canvas API → `image/webp` (quality: 0.7, maxWidth: 1024px)。
- **Base64 互轉**: 照片 Blob ↔ Base64 用於 IndexedDB 序列化。

### 4.3 個人設定 API (Phase 4.2 新增)
- `GET /auth/profile`: 查看個人資訊與代理設定。
- `POST /auth/profile`: 更新代理盤點人 (`proxy_user_id`)。

### 4.4 資產搜尋 API (Phase 4.2 新增)
- `GET /assets/?q=keyword&category=1&location=2&status=active`: 進階搜尋資產列表。
- `GET /assets/my`: 查看個人保管資產列表。
